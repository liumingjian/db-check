# 巡检契约冻结说明

冻结日期：2026-03-05  
适用阶段：MVP 开发与联调  
冻结对象：`manifest.json`、`result.json`、`summary.json`

配套 Schema：
1. `contracts/schemas/manifest.schema.json`
2. `contracts/schemas/result.schema.json`
3. `contracts/schemas/summary.schema.json`

## 1. 冻结原则

1. `result.json` 只存事实，不存执行态与分析结论。
2. 执行态统一放 `manifest.json`。
3. 分析结论统一放 `summary.json`。
4. 缺失语义统一映射，禁止各模块自行解释。

## 2. 契约一：manifest.json（执行清单）

## 2.1 最小字段集（冻结）

| 字段 | 类型 | 必填 | 约束/枚举 |
|---|---|---|---|
| `schema_version` | string | 是 | 固定 `1.0` |
| `run_id` | string | 是 | 全局唯一，建议 `<db_type>-<host>-<ts>` |
| `db_type` | string | 是 | `mysql` \| `oracle` |
| `start_time` | string | 是 | RFC3339 |
| `end_time` | string | 是 | RFC3339，`end_time >= start_time` |
| `exit_code` | integer | 是 | `0` \| `10` \| `20` \| `30` |
| `overall_status` | string | 是 | `success` \| `partial_success` \| `failed` |
| `module_stats` | object | 是 | key 为模块名，value 为模块状态对象 |
| `artifacts.log` | string | 是 | `collector.log` 相对路径 |
| `artifacts.result` | string/null | 是 | `result.json` 路径，失败时可为 `null` |
| `artifacts.summary` | string/null | 否 | 分析后回填 |
| `artifacts.report` | string/null | 否 | 报告后回填 |

模块状态对象（`module_stats.<module>`）最小字段：

| 字段 | 类型 | 必填 | 约束/枚举 |
|---|---|---|---|
| `status` | string | 是 | `success` \| `partial` \| `failed` \| `skipped` |
| `duration_ms` | integer | 是 | `>= 0` |
| `error` | string/null | 是 | `status=failed/partial` 时建议非空 |

## 2.2 最小示例（冻结）

```json
{
  "schema_version": "1.0",
  "run_id": "mysql-192.168.1.101-20260305T030000Z",
  "db_type": "mysql",
  "start_time": "2026-03-05T11:00:00+08:00",
  "end_time": "2026-03-05T11:01:10+08:00",
  "exit_code": 10,
  "overall_status": "partial_success",
  "module_stats": {
    "os": {"status": "success", "duration_ms": 3200, "error": null},
    "db_perf": {"status": "partial", "duration_ms": 2600, "error": "p_s digest truncated"}
  },
  "artifacts": {
    "log": "collector.log",
    "result": "result.json",
    "summary": null,
    "report": null
  }
}
```

## 3. 契约二：result.json（事实层）

## 3.1 最小字段集（冻结）

| 字段 | 类型 | 必填 | 约束/枚举 |
|---|---|---|---|
| `meta.schema_version` | string | 是 | 固定 `2.0` |
| `meta.collector_version` | string | 是 | 采集器版本 |
| `meta.db_type` | string | 是 | `mysql` \| `oracle` |
| `meta.db_host` | string | 是 | 目标地址 |
| `meta.db_port` | integer | 是 | `> 0` |
| `meta.db_name` | string | 否 | 可空字符串 |
| `meta.environment` | string | 否 | `production/staging/dev` 等 |
| `meta.timezone` | string | 是 | IANA 时区，如 `Asia/Shanghai` |
| `meta.collect_time` | string | 是 | RFC3339，任务结束时间 |
| `collect_config.sample_mode` | string | 是 | `single` \| `periodic` |
| `collect_config.sample_interval_seconds` | integer/null | 是 | `periodic` 必填，`single` 为 `null` |
| `collect_config.sample_period_seconds` | integer/null | 是 | `periodic` 必填，`single` 为 `null` |
| `collect_config.expected_samples` | integer | 是 | `>= 1` |
| `collect_window.window_start` | string | 是 | RFC3339 |
| `collect_window.window_end` | string | 是 | RFC3339 |
| `collect_window.duration_seconds` | integer | 是 | `>= 0` |
| `os` | object | 是 | 可为空对象，但必须存在 |
| `db` | object | 是 | 可为空对象，但必须存在 |

## 3.2 事实层硬约束（冻结）

1. 原始指标值必须机器可读：number/boolean/string(enum)。  
2. 禁止把单位写进值里（例如 `"419 ms"`、`"2.47 GiB"`）。  
3. 样本时间必须在 `collect_window` 范围内。  
4. 不输出执行态字段：如 `module status`、`exit_code`。  

## 3.3 最小示例（冻结）

```json
{
  "meta": {
    "schema_version": "2.0",
    "collector_version": "1.2.0",
    "db_type": "mysql",
    "db_host": "192.168.1.101",
    "db_port": 3306,
    "db_name": "order_db",
    "environment": "production",
    "timezone": "Asia/Shanghai",
    "collect_time": "2026-03-05T11:01:10+08:00"
  },
  "collect_config": {
    "sample_mode": "single",
    "sample_interval_seconds": null,
    "sample_period_seconds": null,
    "expected_samples": 1
  },
  "collect_window": {
    "window_start": "2026-03-05T11:01:10+08:00",
    "window_end": "2026-03-05T11:01:10+08:00",
    "duration_seconds": 0
  },
  "os": {},
  "db": {}
}
```

## 4. 契约三：summary.json（分析视图层）

## 4.1 最小字段集（冻结）

| 字段 | 类型 | 必填 | 约束/枚举 |
|---|---|---|---|
| `schema_version` | string | 是 | 固定 `1.0` |
| `run_id` | string | 是 | 必须与 `manifest.run_id` 一致 |
| `rule_version` | string | 是 | 规则版本 |
| `generated_at` | string | 是 | RFC3339 |
| `overall_risk` | string | 是 | `low` \| `medium` \| `high` |
| `counts.total_checks` | integer | 是 | 来自规则总检查项 |
| `counts.normal` | integer | 是 | `>= 0` |
| `counts.warning` | integer | 是 | `>= 0` |
| `counts.critical` | integer | 是 | `>= 0` |
| `counts.unevaluated` | integer | 是 | `>= 0` |
| `counts.not_applicable` | integer | 是 | `>= 0` |
| `abnormal_items` | array | 是 | 告警+严重项 |
| `unevaluated_items` | array | 是 | failed/skipped 导致 |
| `na_items` | array | 是 | not_applicable 项 |
| `failure` | object | 否 | 当 `manifest.exit_code` 为 `20/30` 时必填 |

计数一致性约束（冻结）：

`total_checks = normal + warning + critical + unevaluated + not_applicable`

失败摘要对象（`summary.failure`）约束（冻结）：

| 字段 | 类型 | 必填 | 约束/枚举 |
|---|---|---|---|
| `exit_code` | integer | 是 | `20` \| `30` |
| `reason_type` | string | 是 | `collector_failed` \| `precheck_failed` |
| `message` | string | 是 | 非空 |

失败摘要附加约束（冻结）：
1. `summary.overall_risk` 固定为 `high`（失败保守原则）。  
2. `summary.counts.normal/warning/critical` 必须为 `0`。  
3. `summary.abnormal_items` 必须为空数组。  

## 4.2 最小示例（冻结）

```json
{
  "schema_version": "1.0",
  "run_id": "mysql-192.168.1.101-20260305T030000Z",
  "rule_version": "2.0",
  "generated_at": "2026-03-05T11:02:00+08:00",
  "overall_risk": "high",
  "counts": {
    "total_checks": 128,
    "normal": 60,
    "warning": 40,
    "critical": 8,
    "unevaluated": 18,
    "not_applicable": 2
  },
  "abnormal_items": [],
  "unevaluated_items": [],
  "na_items": []
}
```

## 5. 缺失语义映射表（冻结）

适用对象：分析器在评估单个检查项时，遇到指标缺失或不可判定场景。

| 缺失语义 | 触发条件（示例） | `summary` 归类 | 影响风险等级 | 是否进入 `abnormal_items` | 是否进入 `unevaluated_items` | 是否进入 `na_items` |
|---|---|---|---|---|---|---|
| `failed` | 模块执行失败、查询报错、权限不足 | `unevaluated` | 不直接抬升；由 `unevaluated` 占比策略决定 | 否 | 是 | 否 |
| `skipped` | 按策略跳过（如关闭性能开销采集） | `unevaluated` | 不直接抬升；由 `unevaluated` 占比策略决定 | 否 | 是 | 否 |
| `not_applicable` | 场景不适用（如单机无复制） | `not_applicable` | 不计入风险 | 否 | 否 | 是 |

`unevaluated_items` 最小字段建议：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `check_id` | string | 是 | 规则检查项 ID |
| `reason_type` | string | 是 | `failed` \| `skipped` |
| `reason` | string | 是 | 具体原因（报错或策略说明） |
| `source_module` | string | 否 | 来源模块名 |

`na_items` 最小字段建议：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `check_id` | string | 是 | 规则检查项 ID |
| `reason_type` | string | 是 | 固定 `not_applicable` |
| `reason` | string | 是 | 不适用原因 |

## 6. 分析器判定顺序（冻结）

对每个 `check_id`：

1. 检查是否 `not_applicable`。命中则写入 `na_items`。  
2. 检查数据是否 `failed/skipped`。命中则写入 `unevaluated_items`。  
3. 否则按规则正常判定并写入 `normal/warning/critical`。  

## 7. 退出码与分析行为对齐（冻结）

| `manifest.exit_code` | 含义 | 分析器行为 |
|---|---|---|
| `0` | 采集成功 | 全量评估 |
| `10` | 部分成功 | 允许评估；缺失项按映射表写入 `unevaluated`/`na` |
| `20` | 采集失败 | 不进入常规评估；`summary` 必须输出 `failure` 摘要 |
| `30` | 参数/连接前置失败 | 不进入常规评估；`summary` 必须输出 `failure` 摘要 |

## 8. 冻结变更规则

1. 新增字段：允许（向后兼容）。  
2. 修改/删除冻结字段：禁止，需升级契约版本。  
3. 枚举新增：允许，但必须同步更新分析器校验与文档。  

## 9. 校验命令（本地/CI）

按运行目录校验：

```bash
python3 tasks/validate_frozen_contracts.py --run-dir runs/<run_id>
```

按文件路径校验：

```bash
python3 tasks/validate_frozen_contracts.py \
  --manifest /path/to/manifest.json \
  --result /path/to/result.json \
  --summary /path/to/summary.json \
  --rule /path/to/rule.json

# 严格模式（推荐 CI 使用）
python3 tasks/validate_frozen_contracts.py \
  --manifest /path/to/manifest.json \
  --result /path/to/result.json \
  --summary /path/to/summary.json \
  --rule /path/to/rule.json \
  --strict-schema
```

说明：
1. `--rule` 为可选参数，用于附加校验规则文件结构。  
2. 若环境安装了 `jsonschema`，会执行完整 JSON Schema 校验；未安装时会输出降级告警并继续执行内置强校验。  
3. `--strict-schema` 用于 CI 强门禁，要求 JSON Schema 校验必须可执行。  

本地开发推荐：

```bash
# 样例回放（分析 + 校验）
python3 tasks/local_gate.py sample

# 校验某次运行目录
python3 tasks/local_gate.py run --run-dir runs/<run_id> --rule /path/to/rule.json
```

## 10. Analyzer 映射落地（MVP）

已提供最小分析脚本（内置缺失语义强约束）：

```bash
python3 tasks/analyze_frozen_contracts.py \
  --manifest /path/to/manifest.json \
  --result /path/to/result.json \
  --rule /path/to/rule.json \
  --out /path/to/summary.json
```

可选参数：

```bash
--na-checks /path/to/not_applicable_check_ids.json
```

说明：`--na-checks` 文件为 JSON 数组，内容示例：

```json
["2.4", "2.7", "2.8"]
```
