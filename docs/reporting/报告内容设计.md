# MySQL 巡检报告内容层设计

## 目标

- 以 `docs/reporting/template-mysql.md` 作为章节基准。
- 以统一 `ReportView` 视图模型承载内容，供 Markdown 预览和后续 Word 渲染复用。
- 明确区分 `已采集 / 已推导 / 外部补充 / 未采集 / 不适用`。

## 视图模型

- `ReportView`: 报告根对象，包含标题、生成时间、顶层章节。
- `SectionBlock`: 章节/小节，可递归包含段落、表格、子章节。
- `TableBlock`: 表格块，包含列定义、行数据、状态和说明。

## 章节映射

| 模板章节 | 数据来源 | 状态策略 |
| --- | --- | --- |
| 文档信息 / 修改记录 / 审阅记录 | `meta.json` | `external` |
| 巡检告警定义 | 内置静态文案 | `derived` |
| 巡检范围 | `meta.scope + result.meta + derived role` | `external` |
| 综合健康评估 | `summary.abnormal_items + result` | `derived` |
| 风险发现与整改建议 | `summary.abnormal_items` | `derived` |
| 巡检结论 | `summary.counts + summary.overall_risk` | `derived` |
| 系统指标 | `result.os.*` | `collected` |
| 实例基础信息 | `result.meta + result.db.basic_info + meta.scope` | `external` |
| 主从集群状态 | `result.db.replication + summary.na_items` | `collected/na` |
| mysql关键参数 | `result.db.config_check` | `collected` |
| 数据库容量 | `result.db.storage` | `collected` |
| 数据库用户 | `result.db.security` | `collected` |
| 数据库对象数量 | `result.db.storage.table_index_counts + triggers_procedures_events` | `collected` |
| 运行线程信息 | `result.db.performance` | `collected` |
| 数据库文件信息 | `result.db.storage` | `collected` |
| 数据库状态信息 | `result.db.performance` | `collected` |
| InnoDB / 锁等待 / 元数据锁 / 连接数 | `result.db.performance` | `collected/missing` |
| top 表 / 无主键表 / 非 Innodb / 宽表 / 未使用索引 / 自增值 / 冗余索引 | `result.db.storage` | `collected` |
| 慢 SQL / 全表扫描 SQL | `result.db.sql_analysis` | `collected` |
| 备份策略 / 备份集可用性 | `result.db.backup` | `collected` |
| 模板中暂无 contracts 明细的章节 | 无 | `missing` |

## 当前明确缺口

以下章节会保留在预览里，但按 `missing` 输出，不伪造数据：

- 占用空间 top 10 的索引
- 物理 IO top 10 的表
- 数据库内存分布 top 10
- 全表扫描的表 top10
- 使用临时表的 SQL top10
- 行操作次数 top10
- 单张表超过 6 个索引的对象
- 联合索引字段个数大于 4 的对象

## 输出策略

- Markdown 预览完整保留模板章节层次，用于校对内容设计。
- JSON 视图模型保留结构化层次，用于后续 Word 模版填充。
- 单行参数表统一使用 `参数名称 / 当前值` 列头。
- 文档控制和巡检总结章节保持固定全称表头。
- 巡检明细章节中，列数较少的记录表保持全称表头；仅宽表使用短字段列名，并在表格下输出字段说明。
- DOCX 输出统一基于 `report-view.json` 和模板样式渲染器实现。
